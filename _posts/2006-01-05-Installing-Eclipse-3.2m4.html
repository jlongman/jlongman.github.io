Installing Eclipse 3.2m4

I was chatting with Nic (hi!) and he was asking if I new about IntelliJ feature X in Eclipse, which prompted me to install eclipse 3.2M4. Except that when I added my eclipse_extensions directory - a third-party directory for holding plugins to ease transitions when you upgrade your eclipse - Eclipse wouldn't start! As a matter of fact, it would disappear without even showing a UI error message!

The error message in the metadata log complained about the actionSet in the legacy action set, but didn't even mention which plugin was guilty! I thought maybe some setting from my 3.1 workspace was conflicting, so I started a new workspace... Same problem, crash without notification. Second, I renamed the eclipse extensions directory so it wouldn't find the plugins at all. Eclipse worked, no problem.

Okay, maybe I chose the wrong root in that directory, since I had an outer eclipse_extensions directory containing a directory named eclipse. Crash, no worky. Hmm. I poked around in the eclipse plugin configurations and noticed that it had parented them the same, even though I had offered the inner and outer directories as separate options, so that definitely wasn't it, but there were no further insights in there except that it had only registered two of the plugins in my extensions directly. Clearly, there's a plugin in my extensions screwing things up.

So I went into the eclipse_extensions plugin, noticed one named 'Dashboard' that I couldn't remember being able to access its functionality, ever, deleted it. Wam, it worked.

There are two eclipse plugins with dashboard in the name that I saw off-hand, one for working with Jira, my favourite (quality!) issue tracking software, which sounds cool and I would like to use, but I couldn't remember having installed it; and another for working with cruisecontrol, my 'favourite' (i.e., only one used) continuous integration platform. The second sounds more likely.

So we'll have to see how it goes...
