Eclipse vs IntelliJ

Over the last two months at the new job I've been using a new (for me) IDE from JetBrains called IntelliJ IDEA, or IntelliJ.  IntelliJ pioneered a lot of the IDE refactoring in the Java world, but they're also a pay solution whereas Eclipse is free, as in beer _and_ as in speech.

Nic sent me a link to an IntelliJ blog entry that is collecting reviews comparing the two.  I may have to write one, later when I'm more familiar with IntelliJ.

In the meantime though, one feature I'm definitely missing in IntelliJ is 'click-highlighting'.

In Eclipse, If I click on a variable then every usage of that variable is highlighted in the document and in the scrollbar for easy scrolling to the relevant location.  And it uses zero screen real-estate - I find IntelliJ a lot less dense than eclipse (which is probably good for beginners, not so much for power-users).

Similarly, if you click on the return term, then the points in the method which return are highlighted.  If you click on an exception, the points where that exception are thrown are highlighted.

These are unbelievably useful for grokking other people's code.
