IntelliJ, OSX and kdiff3

update: Jetbrains refuses to acknowledge any problem and frankly their support is offensive. At least when its free software you can say you get what you pay for.

Any IDE that lets you have merge conflicts in its configuration files and then doesn't allow you to fix those problems from the IDE is stupid. IntelliJ is one of those IDEs. It is the single most frustrating thing I find with that pos (piece of software, of course ;-)). Particularly since it should use the equivalent of the <blink> tag to let you know it happened. Instead it silently puts itself into a state where you can't work for an hour why you try to recover the working state of the configuration files.

This was recently exasperated for me by kdiff3, otherwise a fine merge tool (almost as good as xxdiff), as kdiff3 was apparently saving the merged files with some weird line-endings or encoding. The symptoms were that IntelliJ complained that, 'no content was permitted in the prolog', but it did apparently save NEW content to the configuration file, so it was actually intepreting the file. (!) In the end I took the merged file into another editor and saved the file with Unix line endings and 'UTF8 with no BOM'.

It may actually have been the BOM (Byte-Order-Mark) that killed everything, including compilation on the Linux-hosted build server of Java files, but frankly, I'm now 2 hours behind where I wanted to be so I'll have to leave this for an update.
