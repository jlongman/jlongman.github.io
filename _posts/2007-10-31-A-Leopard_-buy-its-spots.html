A Leopard, buy its spots

I'm off to Spain in a couple of days for 6 weeks, but I broke down to take some time to put Leopard on the laptop.  Actually it installed while I was off at the gym getting some climbing in - who knows when I will get to exercise next?  Sleep when you're dead, etc.

Apparently I was using Apple Partitioning on the new drive and Intel Macs and Leopard require 'GUID Partition Tables' or some such non-sense.  As I'd backed up in the last posting, I wasn't too worried about that, save for the time it would take.
It's not easy to find the partition options in Disk Utility either, the option is buried in menus and combo-boxes.  I erased the drive a couple of times before I found the right option. Here is more information than I got during the Leopard install that may help you.

When that was done I used what is basically the Migration Assistant to move the interesting parts from the backup onto my current drive.  I guess I should check whether my work related materials made it - I doubt it.  Oh, I also had to update AddressBook and a few other things.

I wonder if I hadn't updated AddressBook if I would have kept the bluetooth callerId functionality?   (A downgrade indeed)

My good will towards Apple is ebbing a bit - between the the Java treatment in Leopard - yes Java 5 on Leopard is not only not Java 6, but worse than Java 5 on Tiger - and the iPhone's difficulty in syncing music to more than one machine, being usable as a memory stick, the ringtone debacle, the contracts, DRM, etc... I guess it just means the honeymoon's over.  It's still a better platform than Windows, but something sinister may lurk behind the curtain, and maybe that thing is success.

(Not that I don't understand why Apple would be bitter about Sun paying for Windows Java and not OSX Java - and not that I don't think that some of the Leopard features for Cocoa (on Windows?) and Objective-C aren't cool... )
