Eclipse Expand Selection Trick

I was doing some editing which wasn't quite big enough to justify the regex a larger problem would have required, but still could use some improvement in editing. And thus, a quick little trick for Eclipse:

In Eclipse, Control-left and Control-right stop at 'Camel-Case' boundaries. E.g. if I have a variable named 'aliceTheCamelHasFourHumps', then if I placed my cursor before the lower case 'a' in 'alice', I would have to tap Control-right six times in order to select the entire word.

A quicker way to do it is Control-up - which selects the next largest enclosing element, 'entity' or scope

In the simple example above, one tap of Control-up would select the entire variable, already a huge savings, but it will also do so from any position in the variable. In order to use the Control-left or Control-right, I have to be at the extreme right or left of the variable, or go left to the beginning, delete the left portion, then go right and delete the end portion. An extra keypress. But that's assuming I want to delete. Indeed the worse case is I go to the beginning or end of the word in order to select it and perform some other operation on the variable. (Say, Control-2-r, a local file symbol rename, handy for method- and private-scoped symbols)

Of course, if you tap Control-up a second time, it will grab the next largest scope - e.g. if 'aliceTheCamelHasFourHumps' was actually 'aliceTheCamelHasFourHumps.removeAHump()', then it will select the method (the second part) also. It will continue to grow in scope until it gets to the block level, continuing in blocks to the method, the class, the file. Control-down retreats the selection to a lower scope.

Very handy.
