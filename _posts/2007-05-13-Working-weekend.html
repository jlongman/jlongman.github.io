Working weekend

I spent most of the weekend working, but today I was a bit like jello left in the sun too long, so I did some cleaning and upgraded the MythTV frontend (only, not the recording bit) to R5F1, so hot off the presses there's no changelog yet.

Everything went mostly well - I had to play with the mixer a bit to get sound back, basically unmute and turn up the Front channel. Then get mythwelcome re-configured for startup, and set it up so I can shut down the box with the remote (i.e. add the mythtv user to the sudoer list with the shutdown command).

Everything seems slightly nicer. Tho, my nvidia on-board videocard wasn't happy decoding live HDTV so I had to go back to software decoding - watch the CPU jump from 20% usage to 68% usage. I'll have to see if I had a tainted driver in there before - I didn't even bother checking here yet tho.

Finally, while watching the start of the Detroit-Anaheim hockey game, I thought I'd see if the compiler or headers would be anymore compatible with my Antec Fusion VFD. I started downloading the lcdproc sources in order to build it, when it occurred to me this new release may have it already.

This wiki page detailed the changes I needed to make to LCDd.conf (which I think was my old version, as it was missing the trailing slash referenced in the comments!?), modprobe lirc_imon, echo 'woo!' > /dev/lirc0 and bang, it worked. Cool! (Yes, I did it in that order, but there's no point in editing LCDd.conf until the echo works.)

Now start LCDd, get the welcome message, enter the MythTV setup, turn it on and BOOM! it works.

Cool. Of course, it'd be nice if the machine didn't need to be on in order to show me what the backend is doing. The SqueezeBox looks like it could do that...
