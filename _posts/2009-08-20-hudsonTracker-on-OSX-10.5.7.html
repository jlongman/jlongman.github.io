hudsonTracker on OSX 10.5.7

When faced with a Java app that won't start like this:

20/08/09 4:24:27 PM [0x0-0x165b65a].hudsonTracker[97838] [JavaAppLauncher Warning] Java application launched from PPC or bad stub. Relaunching in 32-bit, and tagging sub-processes to prefer 32-bit with $JAVA_ARCH=i386.
20/08/09 4:24:27 PM [0x0-0x165b65a].hudsonTracker[97838] [JavaAppLauncher Error] This process is [i386] and was re-exec'd from [i386], but for some reason we are trying re-exec to [].
20/08/09 4:24:27 PM [0x0-0x165b65a].hudsonTracker[97838] [JavaAppLauncher Error] unable to find a version of Java to launch
20/08/09 4:24:27 PM com.apple.launchd[313] ([0x0-0x165b65a].hudsonTracker[97838]) Exited with exit code: 1

The solution is to copy the /System/Library/Frameworks/JavaVM.framework/Resources/MacOS/JavaApplicationStub over the existing Foo.app/Contents/MacOS/JavaApplicationStub. A more explicit explanation is that the JavaApplicationStub you are copying over does not have the required architecture and the .app is not exposing it so you can set the 32-bit flag (I suspect). If you do 'file Foo.app/Contents/MacOS/JavaApplicationStub' and compare that with 'file /System/Library/Frameworks/JavaVM.framework/Resources/MacOS/JavaApplicationStub', you should see the difference.

I don't know, could be bat guano for all I have time to care. This makes it work.
