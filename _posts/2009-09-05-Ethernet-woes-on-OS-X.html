Ethernet woes on OS X

Seriously - do an OS update and your ethernet stops working...! Claims the cable isn't plugged in.  (I should point out that this is a Leopard, a point update, and not a Snow Leopard upgrade)

But it's the same cable, it used to work, and oh, my Xbox 360 still works using it. Hmmm... This sounds suspicious. A quick search shows the problem is not uncommon and the general solution is to shorten the cable, cause OS X is — apparently — 'picky'. Yes, I am running a long cable. I can't shorten the cable - this is where I need to access the remote boxes from (my apartment is long).

Here's the compromise that worked for me. Go to the Ethernet device in Network settings in System Preferences (click on the Airport radiating fan, last option), choose Advanced, then the last tab, which should be Ethernet. Change the combobox value for Configure: to 'Manually'. Choose the top option (should be 10BT or 10BaseT or something like that). Hit OK, then Apply. See if you get an IP and it starts to work (it did for me). Try higher options until it goes red again. (Only 10BaseT worked for me, so nothing higher ever worked). I also tried to fiddle with the duplex options but it didn't seem to work.

Maybe it was auto-negotiated down to a 10Mbit/s before, but now it's not auto-negotiating it.
