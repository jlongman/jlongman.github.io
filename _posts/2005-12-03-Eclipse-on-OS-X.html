Eclipse on OS X

Do you know what pisses me off about the OS X port of Eclipse? I go to all the trouble of getting a happy hacking keyboard, so I can put the Control key in the right place - and the bloody bastards move the Control key to the Propeller key.

(Which is where the Windows key is, for you Intel people)

And even so, it's half-assed. You know why? Because Spotlight has Propeller-Space, which means Control-Space (y'know, auto-think, i.e. Content Assist) can't be mapped to Propeller-Space.

So what else hasn't been mapped to Propeller?

key Category Command
^Q Navigate Last Edit Location
^D Run/Debug EOF
^H Search Open Search Dialog

which is,
Propeller-Q quits the application.
Propeller-D, dunno?
Propeller-H hides the window.

The Windows version doesn't use the Windows key, you could imagine the problems that would cause. So... if you can't do a proper job with Propeller, and I can't see why you can't stay with Control, why did they do it?

(I think the real mistake was using Propeller-keys as global keys)
