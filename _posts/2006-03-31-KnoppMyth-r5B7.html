KnoppMyth r5B7

So...

Well, I finally gave up on trying to force my hardware encoding card to be the first card. Knoppmyth wanted it to be second to the weaker software encoding card, fine - life is too short. I think the basis for this is that the hardware card requires firmware from the host and the system which provides the ivtv firmware, hotplug, does not start until later in the bootup sequence.

This feels like there are some aspects of the Linux kernel that are beyond my area of experience. And I don't really feel like taking a lot of time to find out about them.

I upgraded to ivtv-0.4.4 as suggested on the Knoppmyth forums, as my digital card was not working. The bad news is Myth doesn't seem to be tuning the internal tuner to channel 3 - so I have to run ivtv-tune -d /dev/video1 -c 3. This is specified in the MythTV Setup program - I don't know why I need to run it manually. I've set it to run when the kernel module is inserted; I can live with that if it works - I have to test it when next nothing is recording.

I rebuilt my remote control (lirc) files - works really well actually. I'm impressed. Of course, mark's braindump has become basically unnecessary since his patches have been integrated into modern ivtv and lirc implementations. He still has some handy scripts and codes though.

Next simple project is adding a coupla hunnerd megs. As it is, we have things we mean to watch, but haven't yet, sitting there consuming hard drive space.
