Eclipse and CVSNT

This one frustrated me for so long, and tonight I was reminded about how awesome Eclipse is when it works. CVSNT exhibits some very Unix'uous behaviour in that it is case sensitive to the path, in a way that a Unix-based CVS is not. End result, Eclipse's CVS support is maimed unless you get the path right.

You don't need to make the repository path look like a Unix path, you just need to make sure that you get the capitalization right. E:/Data/cvs/, not e:/data/cvs. you'll have to get the correct path from the CVS administrator.

If you did check it out wrong, it's not too hard to fix â€“ Go to the CVS Repository View, New-> Repository Location, create the correct path. Then go back to Package explorer, and right-click on your project and edit it's Properties->CVS->Change Sharing. Uncheck the 'Compatible CVS Repositories' button if you don't see it. Double click on your correct repository and repeat until all your projects are done. Once all references to the wrong repository are changed, you can then go back to the CVS Repositories Perspective and delete the wrong reference. (This is the same way you would change the user for a repository).

Then you can browse your Resource History, use the Commit Sets in the Team Synchronizing perspective (a favourite of mine!), and do version diffs and so on. Eclipse rocks!
